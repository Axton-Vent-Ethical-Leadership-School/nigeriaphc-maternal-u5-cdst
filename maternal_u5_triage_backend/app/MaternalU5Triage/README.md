# MaternalU5Triage AI-Enabled Clinical Decision Support Tool

## Overview
MaternalU5Triage is an AI-enabled clinical decision-support system designed to assist primary care triage and referral optimization for maternal and under-five health in rural Nigeria. 

The project is aligned with the grant titled:  
**"Evaluating an AI-Enabled Clinical Decision Support Tool for Primary Care Triage and Referral Optimization in Rural Nigeria"** and focuses on **early-stage validation (Pathway A)**.  

This repository contains all backend and frontend code, documentation, and supporting materials needed for simulation-based evaluation and prototype testing.

- ## 3. Training and Validation

The MaternalU5Triage AI system employs a rigorous model training and validation framework to ensure reliable prediction of maternal and under-five health risks. The framework uses standardized machine learning practices to reduce bias, prevent overfitting, and improve generalization across diverse populations.

---

### Dataset Splitting

The integrated dataset is divided into three subsets to support robust model development and evaluation:

- **Training dataset (70%)**  
  Used to train the machine learning models and allow the algorithm to learn patterns within the data.

- **Validation dataset (15%)**  
  Used to tune model parameters and optimize hyperparameters during training.

- **Testing dataset (15%)**  
  Used for final evaluation of model performance on unseen data to assess real-world predictive capability.

This structured partitioning ensures that the trained models generalize effectively and do not memorize the training data.

---

### Cross-Validation

To further enhance model robustness, the system applies **k-fold cross-validation** during the training process.  

In this approach:

1. The training dataset is divided into k equal subsets.
2. The model is trained on k-1 subsets.
3. The remaining subset is used for validation.
4. This process is repeated k times so each subset serves as the validation set once.

The **average performance** across all folds is used as the final validation score.  

**Benefits of cross-validation:**
- Prevent overfitting  
- Improve model stability  
- Ensure reliable performance across different population samples

---

### Model Evaluation Metrics

The performance of the MaternalU5Triage AI models will be evaluated using multiple statistical metrics commonly used in medical machine learning research:

- **Area Under the Receiver Operating Characteristic Curve (AUROC)**  
  Measures the ability of the model to distinguish between high-risk and low-risk maternal or child health outcomes. A higher AUROC indicates better classification performance.

- **Precision**  
  Measures the proportion of predicted high-risk cases that are truly high risk. High precision ensures that alerts generated by the triage system are clinically meaningful.

- **Recall (Sensitivity)**  
  Measures the ability of the model to correctly identify actual high-risk maternal or neonatal cases. High recall is essential for early detection of complications and reducing preventable deaths.

- **F1-Score**  
  Balances precision and recall, providing a single performance metric useful when dealing with imbalanced health datasets.

---

### Model Selection and Optimization

During training, multiple algorithms may be evaluated, including:

- Random Forest classifiers  
- Gradient Boosting models  
- Deep Neural Networks  

Hyperparameters will be optimized using validation results to achieve the best predictive performance.

---

### Model Validation for Clinical Relevance

To ensure practical usefulness, model outputs will be evaluated in collaboration with maternal and child health experts. The system will be tested using real-world datasets to confirm that predicted risk scores correspond to clinically meaningful outcomes.
